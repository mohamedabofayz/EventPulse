# تقرير الهوية البصرية للمشروع

## نظرة عامة
هذا التقرير يوضح التنسيقات والألوان المستخدمة في مشروع لوحة التحكم لإدارة الغرف الفندقية، لتمكين تطبيق هذه الهوية البصرية في مشاريعك المستقبلية.

## الألوان الأساسية (Color Palette)

### الألوان الرئيسية:
- **اللون الأزرق الداكن**: `#0A2E5A` - يستخدم كلون أساسي للعناوين والأزرار والحدود
- **اللون الأخضر**: `#27AE60` - يستخدم للأزرار الناجحة والتنبيهات الإيجابية
- **اللون البرتقالي**: `#F39C12` - يستخدم لأهمية متوسطة أو تحذيرات
- **اللون الأصفر**: `#F1C40F` - يستخدم للمعلومات أو الحالات المتوسطة
- **اللون الأحمر**: `#E74C3C` - يستخدم للأخطاء أو التنبيهات السلبية
- **اللون الأزرق الفاتح**: `#E8F4FD` - يستخدم كخلفية للبطاقات أو العناصر البارزة

### الألوان الثانوية:
- **اللون الرمادي**: `#6C757D` - يستخدم للنصوص الثانوية
- **لون الحدود**: `#dee2e6` - لتحديد الحدود بين العناصر
- **لون الخلفية العامة**: `#F0F4F8` - خلفية الصفحة الرئيسية
- **لون الخلفية الم-gradient**: `linear-gradient(135deg, #f5f7fa 0%, #c3cfe2 100%)` - في صفحة البداية

## أنماط التصميم

### التنسيقات العامة:
- **نظام الشبكة (Grid)**: يستخدم Grid Layout بشكل واسع مع أنماط مثل `repeat(auto-fit, minmax(...))`
- **Flexbox**: مستخدم لتنظيم العناصر داخل البطاقات والأقسام
- **Responsive Design**: يدعم مختلف أحجام الشاشات باستخدام media queries

### تنسيقات البطاقات (Cards):
- **الإطار**: `border-radius: 16px;` مع `box-shadow` خفيفة
- **الظل**: `box-shadow: 0 4px 20px rgba(0, 0, 0, 0.05);`
- **الحواف**: `border: 1px solid white;`
- **التحويم**: تأثيرات تحريك ورفع البطاقة عند التحويم

### تنسيقات الأزرار:
- **Primary Button**:
  - `background: var(--dark-blue)`
  - `color: white`
  - `border-radius: 8px`
  - `box-shadow: 0 4px 15px rgba(10, 46, 90, 0.3)`
  - تأثير hover: `transform: translateY(-2px)`

- **Success Button**:
  - `background: var(--green)`
  - `color: white`

- **Danger Button**:
  - `background: #fff`
  - `color: var(--danger)`

### حقول الإدخال:
- `background: var(--input-bg)` (`#f8f9fa`)
- `border: 1px solid var(--border)` (`#dee2e6`)
- `border-radius: 8px`
- تأثير focus: `border-color: var(--dark-blue)` مع ظل خارجي

## الخطوط

- **الخط الأساسي**: `'Sans-serif', Arial, sans-serif`
- **حجم العناوين**: يستخدم `clamp()` لضبط الحجم حسب شاشة العرض
- **fontWeight**: تتراوح من 600 إلى 800 للعناصر البارزة

## التأثيرات الحركية (Animations)

- **Fade-in-up**: تأثير تدريجي للظهور من الأسفل
- **Hover effects**: تأثيرات تنقل وتحويم على البطاقات والأزرار
- **Transition**: معظم العناصر لها انتقالات سلسة (`transition: all 0.3s ease`)

## تنسيقات خاصة

### Heatmap (خريطة الحرارة):
- **Very High**: `--very-high: #e74c3c` (أحمر)
- **High**: `--high: #f39c12` (برتقالي)
- **Medium**: `--medium: #f1c40f` (أصفر)
- **Neutral**: `--neutral: #ffffff` (أبيض)

### Switch Toggle:
- تصميم حديث بتأثير انزلاقي
- لون الخلفية عند التفعيل: `var(--green)`

### Modals:
- خلفية شفافة `rgba(0, 0, 0, 0.6)`
- محتوى مركزي مع `border-radius: 16px`
- `box-shadow: 0 20px 50px rgba(0, 0, 0, 0.2)`

## ملاحظات للتطبيق في المشاريع المستقبلية

1. **الاتساق**: استخدام المتغيرات الجذرية لضمان الاتساق عبر جميع صفحات التطبيق
2. **القابلية للتحديث**: يمكن تغيير اللون في مكان واحد (المتغير الجذري) ويتم تحديثه في كل مكان
3. **الدعم الثنائي الاتجاه**: يوجد دعم للغات التي تكتب من اليمين إلى اليسار (RTL)
4. **التحويم والتفاعل**: تأثيرات التحويم تعزز تجربة المستخدم
5. **التصميم المتجاوب**: التنسيقات مصممة لتكون متوافقة مع مختلف أحجام الشاشات

## الاستنتاجات

هذا المشروع يتبع نظام تصميم موحد يعتمد على:
- لوح ألوان متسق قائم على الأزرق الداكن والأخضر
- استخدام متغيرات CSS لتوفير سهولة في الصيانة
- تركيز على تجربة المستخدم من خلال التأثيرات الحركية
- تصميم متجاوب يدعم جميع أحجام الشاشات
- تنسيقات بصرية موحدة عبر جميع الصفحات

يمكنك استخدام هذا النظام كأساس لمشاريعك المستقبلية من خلال نسخ المتغيرات الجذرية واستخدام نفس أنماط التصميم والتنسيقات.
